
# POV Display Refactored (UA)

> Документація апаратної архітектури та протоколу інтеграції (SOP) для матричного поворотного дисплея (Persistence of Vision).

## Коротко про проєкт

Реверс-інжиніринг та пусконалагодження програмно-апаратного комплексу в умовах неповних специфікацій. Проєкт включає збірку мікроелектроніки, вирішення проблем апаратної синхронізації та локалізацію HEX-масивів для виводу кирилиці на LED-матрицю.

*Примітка щодо вихідних даних: Програмне забезпечення, надане постачальником, має закритий вихідний код (Proprietary Software) і не має ліцензії на відкрите публічне поширення. Крім того, оригінальні архіви фізично видалені з серверів дистриб'ютора після закінчення терміну зберігання. Цей репозиторій фокусується виключно на апаратній архітектурі та методології апаратно-програмної інтеграції (Hardware-Software Bridging).*

## Апаратна архітектура (Hardware)

Система побудована на базі двох мікросхем.

| Компонент   | Маркування   | Призначення                                                                                                                                                                                                                        |
| :------------------- | :--------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Головний MCU | `STC12C532S2-LQPF48` | Управління світлодіодною матрицею, обробка переривань, апаратна синхронізація тексту з обертанням двигуна.                                       |
| Модуль RTC     | `DS1302`             | Мікросхема годинника реального часу. Забезпечує збереження та незалежний відлік часу і дати при відключенні основного живлення. |

## Протокол апаратної ініціалізації (SOP)

Підключення для прошивки здійснюється через зовнішній USB-to-TTL конвертер.

**Специфіка прошивки мікроконтролерів серії STC:**
Мікроконтролер `STC12C532S2` не підтримує гаряче прошивання. Для успішного перехоплення завантажувача (bootloader) необхідно виконати жорстке апаратне скидання:

1. Ініціювати процес запису у програмному флешері (фіксація частоти на `18.432 MHz`).
2. Фізично розірвати та відновити ланцюг живлення (`VCC 5V`) плати дисплея.
3. Запис розпочнеться автоматично після виявлення power cycle.

## Реверс-інжиніринг матриці (Кирилична локалізація)

Апаратна специфіка розгортки бокової (вертикальної) панелі дисплея має інвертований вектор рендерингу: символи відображаються у дзеркальному вигляді та у зворотному порядку.

Для забезпечення коректного виводу україномовного тексту застосовано алгоритм подвійної інверсії:

1. **Інверсія стрінгів:** Текстові масиви формувалися у зворотному напрямку перед компіляцією коду.
2. **Апаратне віддзеркалення:** Піксельні матриці кириличних символів проходили віддзеркалення по осі Y у генераторі шрифтів перед експортуванням у фінальний HEX-файл.
